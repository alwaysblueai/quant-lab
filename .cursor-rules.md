# Cursor Project Rules

## Quant research constraints
- Never use future data.
- Never leak labels into features.
- Always explain timestamp alignment.
- Always specify train/validation/test split.
- Always state transaction cost and slippage assumptions.

## Engineering constraints
- Reusable code must go under `src/alpha_lab`.
- Every reusable function should have at least one test.
- Prefer small explicit functions over long scripts.
- Do not silently mutate input dataframes.
- Explain every nontrivial design choice.

## Workflow
- Before writing code, first propose:
  1. objective
  2. function signatures
  3. edge cases
  4. test plan
- After writing code, always run:
  - `uv run ruff check .`
  - `uv run mypy src`
  - `uv run pytest`
